# Contributing to BraveQR

First off, thank you for considering contributing to BraveQR! It's people like you that make BraveQR such a great tool.

## How Can I Contribute?

### Reporting Bugs
- Use the GitHub Issues tracker.
- Describe the exact steps to reproduce the bug.
- Mention your OS and Brave version.

### Suggesting Enhancements
- Open a GitHub Issue with the tag `enhancement`.
- Explain why the feature would be useful.

### Pull Requests
1. **Fork** the repository.
2. **Create a branch** for your feature (`git checkout -b feature/amazing-feature`).
3. **Commit your changes** with descriptive messages.
4. **Push to the branch** (`git push origin feature/amazing-feature`).
5. **Open a Pull Request**.

## Tech Standards
- Use **Manifest V3**.
- Maintain the **Glassmorphism** UI aesthetic.
- Avoid adding heavy external libraries; prefer native APIs where possible.
- Ensure all logic remains **client-side only** (Privacy First).

## Development Setup
The project has no build step. Simply modify the files in the root and reload the extension in `brave://extensions`.
